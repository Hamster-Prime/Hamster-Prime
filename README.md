---

I'm a builder who works across **network infrastructure** and **AI automation**.

I started with MikroTik routers and Shell scripts, and somehow ended up building Telegram bots that think for themselves.

Now I spend most of my time asking: *"Can I make a bot handle this instead?"* — and the answer is usually yes.

---

## What I Do

- 🔧 **Network infra** — RouterOS scripting, BIRD, DNS splitting, OSPF tunnels. I like making packets go where they should.
- 🤖 **Telegram bots** — Group management, content moderation, two-way chat. I've built bots that moderate, teach, and even remember.
- 🧠 **LLM integration** — Not just prompt-and-hope. I build decision engines, vector knowledge bases, and multi-provider fallback chains.
- 🏠 **Home automation** — Home Assistant custom components. NFC tags, Siri shortcuts, making my home just a little smarter.
- 🎧 **Audio & media** — TTS servers with multiple providers, voice synthesis, open-source reading tools.

---

## How I Think

I don't build things to show off. I build things because they need to exist.

Every project starts the same way: I hit a problem that's too repetitive, too slow, or too annoying to deal with manually. Then I automate it.

The best code I write is the code I forget is running.
